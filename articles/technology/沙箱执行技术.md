## 三级沙箱防御体系

**<img src="https://wynneyin.oss-cn-hangzhou.aliyuncs.com/20260330154532017.png" alt="image.png" style="zoom:50%;" />

1.策略守卫，基于一些简单的匹配，拦截一些高危命令，基于静态规则的
2.逻辑门禁，敏感操作需要得到人类的批准
3.物理隔离，通过 docker 的沙箱机制，隔离高危操作，代码运行在不触及核心宿主机的东西。

## Openclaw的沙箱实现
基于 docker，简单来说，在 terminal 中直接跑起来一个 docker。
- Docker 命令入口：[execDocker](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html) 组装 `docker …` 参数并用 `ProcessBuilder` 直接启动子进程，分别捕获 stdout/stderr，默认 120s 等待，非零退出且未允许失败则抛出 `IOException`，成功时返回 [ExecResult](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html)（包含 stdout/stderr/exit code）。参见 [SandboxDocker.java:34-63](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html)。
- 便捷封装：`execDocker(List<String> args)` 直接走上述逻辑，[allowFailure=true](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html) 用于“查询类”调用时不抛异常。参见 [SandboxDocker.java:65-67](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html)。
- 运行态查询：读取端口映射 [readDockerPort](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html)（调用 [docker port](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html) 解析宿主机端口），容器状态 [dockerContainerState](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html)（`docker inspect -f {{.State.Running}}` 判断存在/运行）。参见 [SandboxDocker.java:72-85](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html) 和 [SandboxDocker.java:123-132](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html)。
- 镜像管理：[dockerImageExists](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html) 和 [ensureDockerImage](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html) 用 `docker image inspect` 判断本地镜像；找不到默认镜像时会自动 `pull debian:bookworm-slim` 并 `tag` 成沙箱默认名。参见 [SandboxDocker.java:90-111](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html)。
- 容器创建参数：[buildSandboxCreateArgs](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html) 按配置拼出 `docker create` 参数，包括标签、只读根、tmpfs、网络/用户、能力裁剪、seccomp/AppArmor、安全选项、DNS/hosts、资源限制和卷绑定等。参见 [SandboxDocker.java:137-203](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html)。
- 配置追踪：[readContainerConfigHash](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html) 通过 `docker inspect` 读取容器标签 [openclaw.configHash](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html)，用于判断容器配置是否匹配当前期望。参见 [SandboxDocker.java:207-218](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html)。
