# 互联网后端全景：一个经典互联网公司有哪些模块

> 来源：B站玄米CS，从后端视角梳理互联网公司的完整模块地图。
> AI 时代，前端在消失，后端边界在模糊——看清全貌比深耕单一模块更重要。

---

## 一、入口层：流量从哪里来

后端系统的交互入口主要有三类：

| 入口 | 说明 | 示例 |
|------|------|------|
| **客户端** | 直接使用产品的终端用户 | Web、iOS/Android、PC 客户端 |
| **机器人（Bot）** | 平台嵌入的自动化服务 | 飞书机器人，提供扩展功能 |
| **第三方** | 外部系统跳转与认证 | 抖音第三方登录、OAuth 认证 |

> 无论哪种入口，进来的都可以视为「流量」。

流量会依次经过：

```
客户端 / 机器人 / 第三方
        ↓
   流量平台（清洗、限流、防攻击）
        ↓
   网关层（监控统计、缓存、服务治理）
        ↓
     后端服务
```

- **流量平台**：识别清洗攻击流量，控制流量大小，高峰限流。自建 vs 阿里云（清洗费用较高）
- **网关层**：后端统一入口，负责接口调用统计、成功率监控、缓存加速、服务健康检查

## 二、核心业务域

### 💰 钱：公司生存之本

| 系统 | 功能 |
|------|------|
| **To B 商家平台** | 服务企业客户，管理商家入驻、资质审核 |
| **对账平台** | 资金流水核对，防止差错 |
| **支付系统** | 对接银行/微信/支付宝，处理抽奖、履约、支付 |

> 规模到一定程度后，To B 和 To C 都不可或缺。还需应对二道贩子囤货、洗钱等灰产行为。

### 👤 用户：增长与转化

用户侧的终极目标：**增长 + 转化率**，依赖营销平台：

- **活动配置**：灵活上线促销活动
- **第三方合作**：联合引流
- **内容审核**：防范诈骗、色情等违规内容（避免「帽子叔叔」整改）
- **用户数据报表**：分析用户行为→驱动运营策略
- **BI 展示**：面向老板和管理层的可视化看板

### 📊 数据：一切决策的基础

数据平台是使用面最广的系统：

| 使用者 | 用途 |
|--------|------|
| 普通用户 | 查看个人数据（如年度报告） |
| 产品经理 | 评估功能效果、绩效 |
| 运营 | 活动收益分析 |
| 老板 | 全局经营决策 |

数据架构层：

- **关系型数据库** / **分析型数据库** / **时序数据库**
- **对象存储**：静态资源、设计稿等
- **CDN**：静态资源分发加速
- **DTS**：数据传输与同步
- **流式处理**：实时数据管道

### 🎬 媒体（部分公司才有）

大多数互联网公司不涉及媒体处理。如果有，主要两类：

- **WebRTC**：远程实时通讯——音视频帧对齐、编解码、MCU/SFU 架构，偏底层
- **信令服务**：控制房间状态——谁在哪个房间、谁有发言权

### 🛡️ 风控系统

近年安全事故频发后的标配模块：

- 内容安全审核（用户名、头像、消息）
- 实名认证（身份证 + 手机号绑定 → 提高违法成本）
- 资金往来监控（异常交易自动封禁）
- 机器审核 + 人工审核双通道

## 三、工程基础设施

### 🔧 CI/CD 流水线

代码到上线的桥梁：

- 提供线下/预发/线上多环境
- 多语言打包环境（Java/Node/Go，Linux/Windows）
- 上线异常告警 + 慢 SQL 监控

### 🖥️ 部署与运维

代码最终要落到真实机器上：

| 方案 | 特点 |
|------|------|
| **ECS（云服务器）** | 阿里云等提供的租赁服务，按需使用 |
| **自建机房** | 成本高，需维护，电费贵 |
| **K8S** | 容器集群管理 |
| **域名管理** | 正式域名是对接腾讯、微信等第三方的硬性要求 |

## 四、后端通用组件

不管去哪家公司，后端开发者都要掌握这些「固定组件」：

| 组件 | 作用 | 典型工具 |
|------|------|----------|
| **编程语言** | 应用层/传输层交互 | Java / Go / Python / Node.js |
| **数据库** | 关系型、非关系型、时序、分析、内存/外存 | MySQL / Redis / ES / ClickHouse |
| **异步任务** | 定时/延时异步操作 | Cron / Scheduler |
| **消息队列** | 系统解耦 + 削峰填谷 | Kafka / RocketMQ / RabbitMQ |
| **对象存储** | 文件/图片/视频存储 | MinIO / OSS / S3 |
| **支付底座** | 对接第三方支付 | 微信支付 / 支付宝（直连银行已极少） |
| **流量网关** | 灰度发布、金丝雀、限流 | Nginx / Kong / APISIX |

> 只要理解这些组件的核心能力，并对其一两项实现有深入理解，用什么语言、在哪家公司做后端，本质没有区别。
